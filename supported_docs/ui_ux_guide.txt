UI/UX Guidelines – E-Shop Checkout

1. Layout & Structure
- Page uses a centered container layout with a max-width of 600px.
- Each page section (Products, Cart, Discount, Shipping, Billing Info, Payment) must be clearly separated using headers.

2. Buttons
- "Add" buttons must be blue (#0066cc).
- "Remove" buttons must use red tones (#cc0000, #990000).
- "Pay Now" button must be green (#00aa00) and visually prominent.
- Disabled state of "Pay Now" must be grey.

3. Error Messages
- Validation errors must be shown in red text.
- Error boxes use:
    - Background: #ffcccc
    - Border-left: 3px solid #cc0000

4. Success Messages
- After a successful payment:
    - Show a green success box (#ccffcc)
    - Text must say: “Payment Successful!”
    - Hide product, discount, and form sections.

5. Forms
- Required fields must display a red asterisk.
- Input fields showing errors should have:
    - Red border (#cc0000)
    - Yellowish background (#ffffcc)

6. Responsive Behavior
- On mobile screens (<768px), all form fields stack vertically.
- Inputs should scale up to comfortable touch-friendly sizes.
